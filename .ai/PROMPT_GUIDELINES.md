# Prompt Guidelines

## Execution Rules for AI Assistants
1. **Never Assume**: If a requirement is ambiguous, explicitly ask the user for clarification.
2. **Follow Governance**: Treat `.ai/` documentation as immutable law.
3. **No Unprompted Implementations**: Do not build features or ML logic that were not explicitly requested.
4. **Halt for Approval**: Never run `git commit` or `git push` unless explicitly instructed to do so in the current prompt.
5. **Cite References**: Cross-reference `.ai/` documents or `docs/` specs when justifying architectural decisions.
